# BoothLiner

BoothLiner는 전시 부스, 팝업스토어, 브랜드 체험존 등 오프라인 현장에서 발생하는 고객 접점을 운영 가능한 데이터와 리드로 연결하는 플랫폼입니다.</br>
BoothLiner는 현장 유입부터 문의, 리드 수집, 후속 운영까지 이어지는 오프라인 engagement workflow를 관리합니다.

단순히 부스 정보를 보여주는 서비스가 아니라, 방문자가 부스를 발견하고, 관심을 남기고, 문의를 보내고, 운영자는 그 반응을 실제 리드와 운영 데이터로 연결할 수 있도록 돕는 것이 BoothLiner의 핵심입니다.

BoothLiner가 풀고자 하는 문제는 명확합니다.

- 방문자는 행사장에서 많은 부스를 짧은 시간 안에 둘러봐야 합니다.
- 운영자는 현장에서 들어오는 문의, 리드, 방문 기록을 체계적으로 관리하기 어렵습니다.
- 행사 이후에도 어떤 방문이 실제 비즈니스 기회로 이어졌는지 추적하기 어렵습니다.

BoothLiner는 이 흐름을 하나의 제품 경험으로 연결합니다.

- 방문자는 QR을 스캔하고 부스를 탐색합니다.
- 관심 있는 부스를 저장하고 문의를 남깁니다.
- 운영자는 문의와 리드를 빠르게 확인하고 응답합니다.
- 팀은 행사 운영 결과를 데이터로 확인하고 후속 액션으로 연결합니다.

이를 위해 BoothLiner는 하나의 서비스이지만, 성격이 다른 여러 제품으로 구성됩니다.

## Repositories

### `boothliner-web`

`boothliner-web`은 BoothLiner의 첫인상을 만드는 공개 웹 제품입니다.

이 레포는 BoothLiner를 처음 접하는 사용자를 위한 채널입니다. 브랜드를 보여주고, 행사와 부스를 소개하고, 사용자가 자연스럽게 탐색을 시작할 수 있도록 돕는 역할을 합니다. 검색 유입, 공유 링크, 공개 페이지 경험이 중요한 제품이기 때문에 가장 바깥쪽 사용자 접점을 담당합니다.

이 레포의 특징:

- 공개 사용자 중심의 제품
- 브랜드, 신뢰감, 탐색 경험이 중요함
- SEO와 링크 공유가 중요함
- 행사/부스 발견과 진입 경험을 책임짐
- 웹에서 시작하는 visitor 흐름의 출발점 역할을 함

`boothliner-web`은 운영 도구가 아니라, BoothLiner의 외부 사용자 경험을 담당하는 제품입니다.

### `boothliner-admin`

`boothliner-admin`은 운영자와 내부 팀을 위한 백오피스 제품입니다.

이 레포는 부스를 실제로 운영하는 사람들의 업무 효율을 높이기 위한 도구입니다. 방문자에게 보이는 경험보다, 운영자가 얼마나 빠르고 정확하게 정보를 관리할 수 있는지가 더 중요합니다. 문의 확인, 리드 관리, 부스 설정, 팀 협업, 운영 데이터 확인 같은 기능이 이 제품의 중심이 됩니다.

이 레포의 특징:

- 운영자와 내부 팀 중심의 제품
- 업무 처리 속도와 정확성이 중요함
- 데이터 관리, 권한, 설정, 분석 기능이 핵심
- 방문자 경험보다 운영 생산성이 우선됨
- 풀 기능 백오피스 역할을 담당함

`boothliner-admin`은 BoothLiner의 운영 엔진에 가까운 제품입니다.

### `boothliner-mobile`

`boothliner-mobile`은 현장에서 바로 사용되는 모바일 제품입니다.

BoothLiner의 모바일 앱은 방문자 전용 앱이 아닙니다. 방문자도 사용하고, 운영자도 사용합니다. 방문자는 행사장 안에서 더 빠르게 부스를 탐색하고 반응할 수 있어야 하고, 운영자는 이동 중에도 문의를 확인하고 리드를 관리하며 현장 대응을 할 수 있어야 합니다.

즉 이 레포는 웹을 단순히 줄여놓은 버전이 아니라, 현장성과 즉시성이 중요한 BoothLiner의 모바일 경험을 담당합니다.

이 레포의 특징:

- visitor와 operator가 함께 사용하는 role-based mobile product
- 현장에서 바로 꺼내 쓰는 속도와 단순한 흐름이 중요함
- push notification, QR scan, camera 같은 모바일 기능이 핵심
- 운영자에게는 현장 대응 도구 역할을 함
- 방문자에게는 빠른 탐색과 반응 채널 역할을 함

`boothliner-mobile`은 BoothLiner의 현장 접점을 담당하는 제품입니다.

### `boothliner-backend`

`boothliner-backend`는 BoothLiner의 공통 비즈니스 플랫폼입니다.

이 레포는 `web`, `admin`, `mobile`이 공통으로 사용하는 핵심 도메인과 데이터를 관리합니다. 사용자가 어떤 채널로 들어오든, 결국 부스, 행사, 문의, 리드, 알림, 권한 같은 핵심 로직은 하나의 백엔드에서 일관되게 처리되어야 합니다.

BoothLiner에서 백엔드는 화면을 보여주는 역할이 아니라, 서비스의 규칙과 흐름을 유지하는 역할을 합니다.

이 레포의 특징:

- 모든 클라이언트가 공통으로 의존하는 핵심 플랫폼
- 인증, 권한, 데이터 정합성을 책임짐
- 부스/행사/문의/리드/설문/알림의 중심
- 여러 채널에서 같은 비즈니스 규칙을 일관되게 적용함
- 비동기 작업과 외부 연동의 기준점 역할을 함

`boothliner-backend`는 BoothLiner 전체를 하나의 서비스로 묶어주는 중심 축입니다.

## How These Repositories Work Together

BoothLiner는 레포가 나뉘어 있어도 결국 하나의 서비스 경험을 만듭니다.

- `boothliner-web`은 사용자를 BoothLiner로 유입시키고 탐색을 시작하게 만듭니다.
- `boothliner-mobile`은 현장에서 더 빠르고 즉각적인 경험을 제공합니다.
- `boothliner-admin`은 운영자가 그 반응을 실제 운영과 성과로 연결하게 만듭니다.
- `boothliner-backend`는 이 모든 흐름이 같은 데이터와 규칙 위에서 움직이도록 만듭니다.

즉, BoothLiner는 여러 채널로 나뉜 제품이지만 목적은 하나입니다.

방문자의 현장 반응을 운영 가능한 데이터로 전환하고, 그 데이터를 실제 비즈니스 기회로 연결하는 것.
